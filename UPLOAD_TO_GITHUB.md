## 🚀 Step-by-Step Upload Instructions

### Method 1: Using GitHub Web Interface (Easiest - 2 minutes)

1. **Go to your repository:**
   - Visit: `https://github.com/b6dh55vmv7-wq/Saltash-AR`
   - Click the **"main"** branch button
   - Click **"Upload files"** (or drag and drop)

2. **Upload all files from this project:**
   - `index.html` - Home page
   - `tour.html` - AR experience
   - `markers.html` - Markers guide
   - `README.md` - Documentation
   - `GITHUB_DEPLOYMENT.md` - Deployment guide
   - `DEPLOY_CHECKLIST.md` - Checklist
   - `DEPLOYMENT.md` - All options
   - `QUICK_START.md` - Quick reference
   - `netlify.toml` - Netlify config
   - `vercel.json` - Vercel config
   - `package.json` - Project metadata
   - `.gitignore` - Git config
   - `.github/workflows/deploy.yml` - GitHub Actions

3. **Commit the changes:**
   - Add commit message: "Add Saltash WebAR Historical Tour files"
   - Click **"Commit changes"**

4. **Enable GitHub Pages:**
   - Go to **Settings** (top right)
   - Click **"Pages"** (left sidebar)
   - Under "Build and deployment":
     - Source: **"Deploy from a branch"**
     - Branch: **"main"**
     - Folder: **"/ (root)"**
   - Click **"Save"**

5. **Your site is live!**
   - Wait 1-2 minutes
   - Visit: `https://b6dh55vmv7-wq.github.io/Saltash-AR`

## 🆘 Troubleshooting

### Site not showing
- Refresh page (Ctrl+F5)
- Check Pages settings (should be "Deploy from a branch")
- Verify branch is "main"
- Wait up to 5 minutes

### Camera not working
- Ensure HTTPS (GitHub provides this)
- Grant camera permissions in browser
- Try different browser
- Use smartphone instead of desktop

### AR markers not detecting
- Ensure good lighting
- Print markers at larger size (15cm+)
- Hold device 20-50cm from marker
- Try different device/browser
